# M1-模块支持与清理说明

## 1. 模块支持（按 `galay-rpc` 规则）

已新增 3 个 C++23 命名模块接口文件：

- `galay-http/module/galay.http.cppm`
- `galay-http/module/galay.http2.cppm`
- `galay-http/module/galay.websocket.cppm`

并在顶层 `CMakeLists.txt` 增加了与 `galay-rpc` 同风格的构建规则：

- 选项：`BUILD_MODULE_EXAMPLES`（默认 `ON`）
- 版本门槛：`CMake >= 3.28`
- 生成器门槛：`Ninja` 或 `Visual Studio`
- 模块目标：`galay-http-modules`

构建命令：

```bash
cmake -S . -B build-mod -G Ninja -DBUILD_MODULE_EXAMPLES=ON
cmake --build build-mod --target galay-http-modules -j
```

示例导入：

```cpp
import galay.http;
import galay.http2;
import galay.websocket;
```

## 2. 项目清理策略

本次将压测/测试生成的日志与临时产物纳入清理与忽略范围，避免污染工作区：

- 根目录日志：`*.log`
- `cert` 目录日志：`cert/*.log`
- 额外构建目录：`build_ssl/`
- 压测生成文件：`files/*.bin`

建议清理命令：

```bash
rm -f *.log cert/*.log files/*.bin
```
