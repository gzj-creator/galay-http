# 05-性能测试

本页记录两类信息：

- 当前可直接复现的 benchmark target、真实源码路径与真实运行命令
- 仓库内已经保存的同环境性能对比快照

其中 target/命令属于当前权威信息；吞吐、延迟和竞品对比数字如果没有在当前会话中重新执行，就一律按“历史快照”处理，不能伪装成当前 HEAD 的自动性能承诺。

## 构建前提

### 明文 benchmark

```bash
cmake -S . -B build \
  -DCMAKE_BUILD_TYPE=Release \
  -DBUILD_BENCHMARKS=ON \
  -DBUILD_MODULE_EXAMPLES=OFF
cmake --build build --parallel
```

### TLS benchmark

```bash
cmake -S . -B build-ssl \
  -DCMAKE_BUILD_TYPE=Release \
  -DGALAY_HTTP_ENABLE_SSL=ON \
  -DBUILD_BENCHMARKS=ON \
  -DBUILD_MODULE_EXAMPLES=OFF
cmake --build build-ssl --parallel
```

## 当前性声明

- `benchmark/CMakeLists.txt` 会把 `benchmark/` 目录下的每个 `.cc` 文件直接生成为同名 target
- 本页中的 target 名与命令按源码重新校对
- 除本次真实执行过的验证命令外，其他 benchmark 结果都应视为历史数据
- `docs/performance/` 继续保留更长篇的历史分析页；本页同时允许收录简明的历史对比快照
- 现有历史文本结果文件位于 `benchmark/results/`

## 同环境性能对比快照

以下内容来自仓库内已经保存的 `benchmark/results/*/summary.md`。它们是当时同环境、同线程配置下的历史快照，适合用于看趋势和回顾优化收益，不代表当前 HEAD 未重跑时的自动承诺。

### 2026-03-22 全协议基线

来源：

- `benchmark/results/20260322-174642-galay-go-rust-http-proto-compare/summary.md`

统一线程数：`4`

| 协议 | Galay | Go | Rust | 备注 |
| --- | ---: | ---: | ---: | --- |
| `http` | `152483.50 rps` | `152144.12 rps` | `155876.25 rps` | 三者接近 |
| `https` | `104343.25 rps` | `87899.00 rps` | `108029.62 rps` | Galay 与 Rust 接近 |
| `ws` | `192434.75 rps` | `189221.12 rps` | `191555.38 rps` | Galay 已具竞争力 |
| `wss` | `138206.50 rps` | `175585.62 rps` | `173214.38 rps` | TLS WebSocket 有差距 |
| `h2c` | `120254.00 rps` | `74534.50 rps` | `114860.88 rps` | 明文 HTTP/2 已较强 |
| `h2` | `2110.38 rps` | `73006.50 rps` | `91475.75 rps` | 当时的主要异常瓶颈 |

这组基线说明：真正的问题集中在 TLS 变体，尤其是 `h2`；`ws` 和 `h2c` 当时已经不是主要短板。

### 2026-03-22 聚焦复跑：`wss` / `h2`

来源：

- `benchmark/results/20260322-200458-galay-go-rust-http-proto-compare/summary.md`

统一线程数：`4`

| 协议 | Galay | Go | Rust | 备注 |
| --- | ---: | ---: | ---: | --- |
| `wss` | `144561.12 rps` | `130321.50 rps` | `130145.88 rps` | 聚焦优化后已反超 |
| `h2` | `78578.88 rps` | `49257.62 rps` | `66376.88 rps` | 聚焦优化后显著恢复 |

这组快照对应的结论是：在当时那轮同环境复跑里，`wss` 与 `h2` 的差距主要来自共享 TLS 热路径和接受连接策略，而不是协议表面 API 本身。

## 验证闭环

建议至少做两类验证：

```bash
# 1) 构建 target 名是否真实存在
cmake --build build --target B1-HttpServer B11-H2cMuxClient B15-HeaderParsing --parallel 4

# 2) 单机 smoke（示例）
./build/benchmark/B1-HttpServer 8080 4
./build/benchmark/B11-H2cMuxClient 127.0.0.1 9080 10 10 10 30 1
```

TLS benchmark 同理，只是需要切换到 `build-ssl/benchmark/` 并启用 `-DGALAY_HTTP_ENABLE_SSL=ON`。
