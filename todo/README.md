# TODO 目录说明

本目录用于管理项目的待办事项列表。

## 使用规范

根据 C++ 开发规范，每次开始新需求前，应在此目录创建待办列表文件。

### 文件命名格式

```
YYYY-MM-DD-功能名称.md
```

例如：`2026-01-29-http-range-support.md`

### 待办事项格式

每个待办事项使用以下格式：

```markdown
- [false] 待完成的任务描述
- [true] 已完成的任务描述
```

### 示例

```markdown
# HTTP Range 支持功能

## 待办事项

- [false] 实现 Range 请求解析
- [false] 实现 Range 响应生成
- [false] 添加单元测试
- [false] 添加压测代码
- [false] 编写文档
- [false] 提交代码

## 完成标准

- 所有单元测试通过
- 压测性能达标
- 文档完整
```

## 工作流程

1. **需求接收**: 创建待办列表，所有事项标记为 `[false]`
2. **代码实现**: 完成功能开发
3. **测试**: 编写并运行测试，通过后标记为 `[true]`
4. **压测**: 运行压测，通过后标记为 `[true]`
5. **文档**: 编写文档，完成后标记为 `[true]`
6. **提交**: 所有事项为 `[true]` 后提交 git

## 注意事项

- 待办列表应该具体、可衡量
- 每个任务完成后立即更新状态
- 保留历史待办列表作为项目记录
